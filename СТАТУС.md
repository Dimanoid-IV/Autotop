# Статус проекта AutoTop

## ✅ Что уже готово

### Основной функционал

1. **Многоязычность (RU/ET)**
   - Полная поддержка русского и эстонского языков
   - URL-маршрутизация (`/ru/` и `/et/`)
   - Файлы переводов для всего интерфейса
   - Переключатель языков

2. **Система аутентификации**
   - Регистрация через email/пароль
   - Вход через Google OAuth
   - Роли пользователей (Гость, Пользователь, Админ, Владелец бизнеса)
   - Защищенные маршруты

3. **Управление бизнесами**
   - Добавление автосервисов (любой пользователь)
   - Профили бизнесов с картами
   - 5 категорий услуг (Автосервис, Авторемонт, Автомойка, Детейлинг, Сервисная станция)
   - Фильтрация по городам
   - Система верификации бизнесов

4. **Система отзывов и рейтингов**
   - 5-звездочная система оценки
   - Публикация отзывов (только для авторизованных)
   - Модерация через email
   - Админ-панель для одобрения/отклонения
   - Ответы владельцев бизнесов (тоже с модерацией)

5. **Пользовательский интерфейс**
   - Главная страница с hero-секцией
   - Липкая поисковая строка
   - Сетка карточек бизнесов
   - Секция "Топ-10 бизнесов"
   - Блок последних отзывов
   - Адаптивный дизайн (мобильные, планшеты, десктоп)

6. **Админ-панель**
   - Интерфейс модерации отзывов
   - Email-уведомления о новых отзывах/ответах
   - Одобрение/отклонение в один клик

### Техническая реализация

- ✅ База данных: Prisma ORM с PostgreSQL (Neon)
- ✅ Backend API: 15+ эндпоинтов
- ✅ Frontend: Next.js 14, React, TypeScript
- ✅ Стилизация: Tailwind CSS
- ✅ Карты: Leaflet (OpenStreetMap)
- ✅ Email: Nodemailer для модерации
- ✅ Аутентификация: NextAuth.js

## 📋 Следующие шаги

### 1. Настройка окружения (обязательно)

```bash
# Установить зависимости
npm install

# Скопировать файл окружения
cp .env.example .env

# Отредактировать .env и добавить:
# - DATABASE_URL (строка подключения к Neon PostgreSQL)
# - NEXTAUTH_SECRET (сгенерировать: openssl rand -base64 32)
# - NEXTAUTH_URL (http://localhost:3000 для разработки)
# - GOOGLE_CLIENT_ID и GOOGLE_CLIENT_SECRET (из Google Cloud Console)
# - SMTP настройки (для email-уведомлений)
# - ADMIN_EMAIL (email для модерации)
```

### 2. Настройка базы данных

```bash
# Сгенерировать Prisma Client
npm run db:generate

# Применить схему к базе данных
npm run db:push

# Заполнить начальные данные (города, категории, админ-пользователь)
npm run db:seed
```

**Важно:** После seed-скрипта создается админ:
- Email: `admin@autotop.ee`
- Пароль: `admin123`
- **Обязательно смените пароль после первого входа!**

### 3. Настройка Google OAuth

1. Перейти в [Google Cloud Console](https://console.cloud.google.com/)
2. Создать новый проект (или выбрать существующий)
3. Включить Google+ API
4. Создать OAuth 2.0 credentials
5. Добавить авторизованный URI перенаправления:
   - Для разработки: `http://localhost:3000/api/auth/callback/google`
   - Для продакшена: `https://yourdomain.com/api/auth/callback/google`
6. Скопировать Client ID и Client Secret в `.env`

### 4. Настройка Email (SMTP)

Для модерации отзывов нужен SMTP-сервер. Варианты:

**Gmail (для разработки):**
- Создать App Password в настройках Google Account
- Использовать:
  - SMTP_HOST: `smtp.gmail.com`
  - SMTP_PORT: `587`
  - SMTP_USER: ваш email
  - SMTP_PASSWORD: App Password

**SendGrid/Mailgun (для продакшена):**
- Зарегистрироваться на сервисе
- Получить SMTP credentials
- Добавить в `.env`

### 6. Тестирование основного функционала

1. **Регистрация/Вход**
   - Зарегистрироваться или войти
   - Проверить Google OAuth

2. **Добавление бизнеса**
   - Перейти в "Добавить услугу"
   - Заполнить данные
   - Сохранить

3. **Оставить отзыв**
   - Перейти на страницу бизнеса
   - Написать отзыв (статус: PENDING)

4. **Модерация отзыва**
   - Проверить email админа
   - Кликнуть "Одобрить" в письме
   - Отзыв появится на странице бизнеса

5. **Ответ владельца**
   - Войти как владелец бизнеса
   - Ответить на отзыв
   - Ответ также требует модерации

## 🆘 Проблемы и решения

**Проблема:** База данных не подключается
- Проверьте DATABASE_URL
- Убедитесь, что SSL включен для Neon
- Проверьте доступность базы

**Проблема:** Google OAuth не работает
- Проверьте Client ID и Secret
- Убедитесь, что redirect URI правильный
- Проверьте, что API включен в Google Cloud

**Проблема:** Email не отправляется
- Проверьте SMTP credentials
- Убедитесь, что SMTP_FROM правильный
- Проверьте логи ошибок

**Проблема:** i18n не работает
- Убедитесь, что locale в URL (`/ru/` или `/et/`)
- Проверьте файлы переводов в `messages/`
- Проверьте middleware
