# Исправление DOM ошибок и 404 на странице бизнеса

## Проблемы

1. **404 на странице бизнеса** - `/ru/businesses/[id]` возвращает 404
2. **DOM ошибки:**
   - `HierarchyRequestError: Only one element on document allowed`
   - `NotFoundError: The node to be removed is not a child of this node`
   - `AggregateError` - множественные ошибки

## Причины

### 1. 404 на странице бизнеса
- Бизнеса нет в базе данных с таким ID
- API `/api/businesses/[id]` возвращает 404
- Неправильный URL или формат ID

### 2. DOM ошибки
- Возможно, Next.js пытается рендерить несколько root элементов
- Проблемы с hydration (несоответствие серверного и клиентского рендеринга)
- Конфликт между root layout и locale layout

## Решения

### 1. Проверьте данные в базе

Выполните в Neon Console:
```sql
SELECT id, name FROM "Business" LIMIT 10;
```

Убедитесь, что бизнесы существуют и имеют правильные ID.

### 2. Проверьте API напрямую

Откройте в браузере:
```
https://autotop.vercel.app/api/businesses/[id]
```
(замените `[id]` на реальный ID из базы)

**Ожидаемый результат:**
- JSON с данными бизнеса ✅
- `404 Not Found` → бизнеса нет в базе
- `500 Internal Server Error` → проблема с API

### 3. Очистите кеш браузера

DOM ошибки могут быть из-за кеша:
1. `Ctrl + Shift + Delete`
2. Выберите "Cached images and files"
3. Выберите "All time"
4. Нажмите "Clear data"
5. Обновите страницу (`Ctrl + Shift + R`)

### 4. Проверьте переменные окружения

Убедитесь, что на Vercel установлены:
- `NEXTAUTH_URL` = `https://autotop.vercel.app`
- `DATABASE_URL` = строка подключения из Neon
- `NEXTAUTH_SECRET` = сгенерированный ключ

## Если проблема сохраняется

### Проверьте логи на Vercel:
1. Vercel → Deployments → выберите деплой
2. Function Logs → ищите ошибки для `/api/businesses/[id]`

### Проверьте консоль браузера:
1. Откройте DevTools (F12)
2. Console tab
3. Скопируйте все ошибки

### Проверьте Network tab:
1. DevTools → Network tab
2. Обновите страницу
3. Найдите запрос к `/api/businesses/[id]`
4. Проверьте статус и ответ

## Важно

Если бизнеса нет в базе данных, страница всегда будет возвращать 404. Убедитесь, что данные загружены в базу.
