# Как задеплоить на Vercel вручную

## Проблема: Автоматический деплой не работает

Если после push на GitHub Vercel не запускает автоматический деплой, вот что нужно сделать:

## Способ 1: Redeploy через Vercel Dashboard (РЕКОМЕНДУЕТСЯ)

1. **Зайдите в Vercel Dashboard:**
   - Откройте https://vercel.com
   - Войдите в свой аккаунт
   - Найдите проект `Autotop`

2. **Откройте Deployments:**
   - В меню проекта выберите **Deployments**
   - Найдите последний деплой (должен быть с вашим последним коммитом)

3. **Сделайте Redeploy:**
   - Нажмите на три точки (⋯) рядом с деплоем
   - Выберите **Redeploy**
   - В появившемся окне можете снять галочку "Use existing Build Cache" (если хотите полную пересборку)
   - Нажмите **Redeploy**

4. **Дождитесь завершения:**
   - Следите за процессом в логах
   - Деплой обычно занимает 1-3 минуты

## Способ 2: Проверьте настройки автоматического деплоя

1. **Зайдите в Settings → Git:**
   - Убедитесь, что репозиторий подключен
   - Проверьте, что выбрана правильная ветка (обычно `main` или `master`)
   - Убедитесь, что включен "Automatic deployments from Git"

2. **Проверьте подключение GitHub:**
   - Settings → Git → Connected Git Repository
   - Должен быть указан: `Dimanoid-IV/Autotop`
   - Если нет - нажмите "Connect Git Repository" и подключите заново

## Способ 3: Создайте новый деплой через GitHub

1. **Сделайте пустой коммит:**
   ```bash
   git commit --allow-empty -m "Trigger Vercel deployment"
   git push origin main
   ```

2. **Или измените что-то в коде:**
   - Добавьте комментарий в любой файл
   - Закоммитьте и запушьте

## Способ 4: Проверьте статус деплоя

1. **Зайдите в Deployments:**
   - Посмотрите на последний деплой
   - Если статус "Ready" - значит деплой прошел успешно
   - Если статус "Building" - дождитесь завершения
   - Если статус "Error" - проверьте логи

2. **Проверьте логи:**
   - Нажмите на деплой
   - Откройте вкладку **Build Logs** или **Function Logs**
   - Ищите ошибки

## Способ 5: Отключите и подключите репозиторий заново

Если ничего не помогает:

1. **Settings → Git → Disconnect**
2. **Settings → Git → Connect Git Repository**
3. Выберите `Dimanoid-IV/Autotop`
4. Настройте проект заново (переменные окружения сохранятся)

## Частые проблемы

### Проблема 1: "No deployments found"
**Решение:** Проверьте, что репозиторий правильно подключен к Vercel

### Проблема 2: Деплой зависает на "Building"
**Решение:** 
- Подождите 5-10 минут
- Если не помогло - отмените деплой и запустите новый

### Проблема 3: Деплой завершается с ошибкой
**Решение:**
- Проверьте Build Logs
- Убедитесь, что все переменные окружения установлены
- Проверьте, что `package.json` содержит правильные скрипты

### Проблема 4: Изменения не появляются на сайте
**Решение:**
- Очистите кеш браузера (Ctrl+Shift+R)
- Проверьте, что деплой действительно завершился успешно
- Убедитесь, что вы смотрите правильный URL (Production, а не Preview)

## Быстрая проверка

1. ✅ Репозиторий подключен к Vercel?
2. ✅ Ветка `main` выбрана для деплоя?
3. ✅ Автоматический деплой включен?
4. ✅ Последний коммит виден в Vercel?
5. ✅ Переменные окружения установлены?

## Если ничего не помогает

1. Создайте новый проект на Vercel
2. Подключите тот же репозиторий
3. Установите переменные окружения
4. Задеплойте
