# Как проверить логи конкретных функций на Vercel

## Шаг 1: Откройте логи проблемных функций

На странице Functions, которую вы видите, нужно проверить логи функций, которые возвращают 500:

### Функции для проверки:

1. **`/api/cities`** - возвращает 500
2. **`/api/categories`** - возвращает 500  
3. **`/api/auth/[...nextauth]`** - возвращает 500

## Шаг 2: Как открыть логи функции

1. На странице Functions найдите функцию `/api/cities`
2. **Кликните на неё** - откроется детальная страница
3. Перейдите на вкладку **Logs** или **Function Logs**
4. Ищите ошибки в логах

## Шаг 3: Что искать в логах

### Для `/api/cities` и `/api/categories`:

Ищите эти ошибки:

#### `DATABASE_URL is not set`
**Проблема:** Переменная окружения не установлена
**Решение:** 
- Settings → Environment Variables → Добавьте `DATABASE_URL`
- Значение: строка подключения из Neon Console
- Выберите окружение **Production**
- Сделайте Redeploy

#### `Prisma Client is not available`
**Проблема:** Prisma не может подключиться
**Решение:**
- Проверьте `DATABASE_URL` - правильный ли формат
- Убедитесь, что база данных активна в Neon

#### `Error fetching cities: ...` или `Error fetching categories: ...`
**Проблема:** Ошибка при запросе к базе данных
**Решение:**
- Проверьте подключение к базе данных
- Убедитесь, что таблицы `City` и `Category` существуют

### Для `/api/auth/[...nextauth]`:

Ищите эти ошибки:

#### `NEXTAUTH_SECRET is not set`
**Проблема:** Переменная окружения не установлена
**Решение:**
- Settings → Environment Variables → Добавьте `NEXTAUTH_SECRET`
- Значение: сгенерированный ключ
- Выберите окружение **Production**
- Сделайте Redeploy

#### `DATABASE_URL is not set`
**Проблема:** Переменная окружения не установлена
**Решение:**
- Settings → Environment Variables → Добавьте `DATABASE_URL`
- Значение: строка подключения из Neon Console
- Выберите окружение **Production**
- Сделайте Redeploy

#### `Failed to initialize NextAuth`
**Проблема:** NextAuth не может инициализироваться
**Решение:**
- Проверьте все переменные окружения
- Убедитесь, что `NEXTAUTH_URL` установлен правильно

## Шаг 4: Проверьте переменные окружения

1. Vercel → Settings → Environment Variables
2. Убедитесь, что установлены:
   - ✅ `DATABASE_URL` (для Production)
   - ✅ `NEXTAUTH_URL` (для Production) 
   - ✅ `NEXTAUTH_SECRET` (для Production)

3. Если переменных нет - добавьте их:
   - `DATABASE_URL` = строка подключения из Neon
   - `NEXTAUTH_URL` = `https://autotop-jb6gh7ghd-dimanoid-ivs-projects.vercel.app`
   - `NEXTAUTH_SECRET` = сгенерированный ключ

## Шаг 5: После установки переменных

1. **Сохраните переменные**
2. **Сделайте Redeploy:**
   - Deployments → последний деплой → три точки (⋯) → Redeploy
3. **Дождитесь завершения**
4. **Проверьте результат:**
   - Откройте сайт
   - Откройте консоль браузера (F12)
   - Обновите страницу
   - Ошибки 500 должны исчезнуть

## Быстрая проверка

Откройте в браузере (без авторизации):

```
https://autotop-jb6gh7ghd-dimanoid-ivs-projects.vercel.app/api/cities
https://autotop-jb6gh7ghd-dimanoid-ivs-projects.vercel.app/api/categories
```

**Ожидаемый результат:**
- `[]` (пустой массив) - если БД пустая, но API работает ✅
- `[{...}, {...}]` (массив с данными) - если все работает ✅

**Если видите:**
- `500 Internal Server Error` → проверьте логи функции
- `{"error": "..."}` → скопируйте ошибку
