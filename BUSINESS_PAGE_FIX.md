# Исправление проблемы с кнопкой "Подробнее"

## Проблема
При нажатии на кнопку "Подробнее" на карточке бизнеса:
- 404 ошибка на `/ru/businesses/[id]`
- Ошибки DOM манипуляции
- Страница бизнеса не открывается

## Что было исправлено

### 1. BusinessCard - правильное использование локали
- Добавлен `useParams()` для получения текущей локали
- Link компонент из `next-intl` автоматически добавляет локаль к пути

### 2. BusinessPage - правильный URL для fetch
- Исправлен способ получения базового URL для серверного компонента
- Используется `NEXTAUTH_URL` или `VERCEL_URL` вместо `window.location.origin`
- Добавлены заголовки для fetch запроса

## Проверка

После деплоя проверьте:

1. **Откройте сайт**
2. **Найдите карточку бизнеса**
3. **Нажмите "Подробнее"**
4. **Страница должна открыться без ошибок**

## Если проблема сохраняется

### Проверьте:
1. Есть ли данные в базе для этого бизнеса
2. Правильно ли работает API `/api/businesses/[id]`
3. Что показывает консоль браузера при нажатии на кнопку

### Проверьте API напрямую:
Откройте в браузере:
```
https://autotop.vercel.app/api/businesses/[id]
```
(замените `[id]` на реальный ID бизнеса)

**Ожидаемый результат:**
- JSON с данными бизнеса ✅
- `404 Not Found` → бизнеса нет в базе
- `500 Internal Server Error` → проблема с API

## Важно

Убедитесь, что:
- ✅ Переменная `NEXTAUTH_URL` установлена на Vercel
- ✅ База данных содержит записи в таблице `Business`
- ✅ API `/api/businesses/[id]` работает правильно
