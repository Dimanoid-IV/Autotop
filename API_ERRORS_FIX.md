# Исправление ошибок API и NextAuth

## Проблемы, которые были исправлены

### 1. Ошибки 500 на `/api/cities` и `/api/categories`
**Проблема:** API возвращали 500 ошибку, если DATABASE_URL не был установлен или произошла ошибка подключения.

**Решение:**
- Добавлена проверка `DATABASE_URL` перед использованием Prisma
- API теперь возвращают пустой массив `[]` вместо ошибки 500
- Это позволяет приложению работать даже если БД недоступна (показывает пустые списки)

### 2. Ошибка NextAuth `CLIENT_FETCH_ERROR`
**Проблема:** NextAuth возвращал пустой ответ или невалидный JSON, вызывая ошибку "Unexpected end of JSON input".

**Решение:**
- Добавлена проверка `NEXTAUTH_SECRET` и `DATABASE_URL` в `createAuthOptions()`
- Добавлена обработка ошибок в GET и POST handlers NextAuth
- Теперь возвращается валидный JSON с ошибкой вместо пустого ответа

### 3. 404 ошибки на несуществующие страницы
**Проблема:** В Header и Footer были ссылки на страницы, которые не существуют:
- `/ratings`
- `/cities`
- `/contacts`
- `/faq`
- `/terms`
- `/privacy`

**Решение:**
- Убраны ссылки на несуществующие страницы из Header
- Убраны ссылки на несуществующие страницы из Footer
- Оставлены только рабочие ссылки: `/` и `/add-service`

## Что нужно проверить на Vercel

### 1. Переменные окружения (КРИТИЧНО!)
Убедитесь, что установлены:
- ✅ `DATABASE_URL` - строка подключения из Neon
- ✅ `NEXTAUTH_URL` - URL вашего приложения (например, `https://autotop-jb6gh7ghd-dimanoid-ivs-projects.vercel.app`)
- ✅ `NEXTAUTH_SECRET` - секретный ключ

### 2. После деплоя проверьте:
1. Откройте консоль браузера (F12)
2. Обновите страницу
3. Ошибки 500 должны исчезнуть
4. Ошибки 404 на `/ratings`, `/cities` и т.д. должны исчезнуть
5. NextAuth ошибка должна исчезнуть (или показывать понятное сообщение)

## Если ошибки остались

### Проблема: Все еще 500 на API
**Решение:**
1. Проверьте Function Logs на Vercel
2. Убедитесь, что `DATABASE_URL` установлен правильно
3. Проверьте, что база данных доступна из Vercel

### Проблема: NextAuth все еще не работает
**Решение:**
1. Проверьте, что `NEXTAUTH_URL` установлен правильно (должен быть полный URL вашего приложения)
2. Проверьте, что `NEXTAUTH_SECRET` установлен
3. Проверьте Function Logs на Vercel для деталей ошибки

### Проблема: Карточки бизнесов не отображаются
**Решение:**
1. Проверьте `/api/businesses` напрямую в браузере
2. Проверьте консоль браузера на ошибки
3. Проверьте, есть ли данные в базе данных
4. См. `TROUBLESHOOTING_BUSINESS_CARDS.md`
