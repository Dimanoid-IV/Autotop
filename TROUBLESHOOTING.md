# Решение проблем

## Ошибка: "Cannot read properties of undefined (reading 'length')"

**Причина:** API возвращает ошибку или неправильную структуру данных, когда база данных не настроена.

**Решение:** 
1. Убедитесь, что база данных настроена (см. QUICK_START.md)
2. Примените схему: `npm run db:push`
3. Заполните данные: `npm run db:seed`

**Временное решение:** Приложение теперь корректно обрабатывает отсутствие данных и показывает "No businesses found" вместо ошибки.

## Ошибка: "Module not found: Can't resolve '../../messages'"

**Причина:** Неправильный путь к файлам переводов в next-intl.

**Решение:** Исправлено в `i18n/request.ts` - теперь используется явный switch/case для всех локалей.

## Ошибка подключения к базе данных

**Причина:** DATABASE_URL не настроен или неправильный.

**Решение:**
1. Создайте файл `.env`
2. Добавьте `DATABASE_URL` с правильной строкой подключения
3. Для Neon PostgreSQL: скопируйте connection string из панели управления

## Приложение запускается, но показывает пустую страницу

**Причина:** База данных пустая или не настроена.

**Решение:**
```bash
npm run db:seed
```

Это создаст:
- Города (Tallinn, Tartu, Narva, Pärnu)
- Категории услуг
- Админ-пользователя (admin@autotop.ee / admin123)

## API возвращает ошибки 500

**Причина:** База данных не настроена или Prisma Client не сгенерирован.

**Решение:**
```bash
npm run db:generate
npm run db:push
```

## Проблемы с аутентификацией

**Причина:** NEXTAUTH_SECRET не установлен.

**Решение:** Добавьте в `.env`:
```
NEXTAUTH_SECRET=любая-случайная-строка-для-разработки
```

## Email не отправляется

**Причина:** SMTP не настроен.

**Решение:** 
- Для разработки можно оставить пустым (модерация не будет работать)
- Для продакшена настройте SMTP (Gmail, SendGrid и т.д.)

## Google OAuth не работает

**Причина:** Google OAuth credentials не настроены.

**Решение:**
1. Создайте проект в Google Cloud Console
2. Включите Google+ API
3. Создайте OAuth 2.0 credentials
4. Добавьте redirect URI: `http://localhost:3000/api/auth/callback/google`
5. Добавьте в `.env`: `GOOGLE_CLIENT_ID` и `GOOGLE_CLIENT_SECRET`

## Страница перезагружается бесконечно

**Причина:** Проблема с middleware или i18n routing.

**Решение:** Убедитесь, что:
- Файлы переводов существуют в `messages/et.json` и `messages/ru.json`
- Middleware правильно настроен
- URL содержит локаль: `/et/` или `/ru/`
